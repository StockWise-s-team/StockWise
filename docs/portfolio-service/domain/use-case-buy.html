Use Case

UC-BUY — Place Buy Order

UC-BUY · Updated 2026-06-02 · Source root services/portfolio-service

Investor submits a paper BUY order for a symbol and receives a Transaction.

UC-BUYuse-casedocumented

Key Facts (SRS)

ActorUser (Nhà đầu tư), Manager, Admin (qua API Gateway)
GoalTiếp nhận và xử lý lệnh MUA cổ phiếu trên Trading Sandbox
TriggerUser nhập thông tin lệnh Mua (Mã CK, Giá, Khối lượng) và nhấn "Đặt lệnh" trên Frontend
Preconditions
  • User đã đăng nhập (JWT hợp lệ, chưa hết hạn).
  • Portfolio Service hoạt động bình thường (health check 200 OK).
  • Message Queue (RabbitMQ) đang kết nối và sẵn sàng nhận message.
  • Database Orders đang kết nối và ghi được.
  • Mã cổ phiếu (ticker) đang trong phiên giao dịch.
Postconditions (success)
  • Bản ghi lệnh mới trong bảng Orders với status = PENDING.
  • Số tiền (Giá × Khối lượng) đã được đóng băng (freeze) trong ví user.
  • Event "ORDER_CREATED" đã được publish lên Message Queue.
  • User nhận HTTP 201 với order_id và status = PENDING.
Postconditions (failure)
  • Không có bản ghi nào được tạo trong database.
  • Không có message nào được publish lên Queue.
  • Không thay đổi số dư ví của user.
  • User nhận HTTP 4xx/5xx kèm thông báo lỗi tương ứng.
Business FlowBFLOW-ORDER-EXECUTION
ExceptionsSRS Order Exception Flows (E1–E10)
TraceabilityTRACE-PORTFOLIO

Main Scenario (SRS)

  1. User nhập thông tin: Mã CK, Loại lệnh = "MUA", Giá, Khối lượng. Nhấn "Đặt lệnh".
  2. Hệ thống kiểm tra JWT Token, xác thực và trích xuất userId.
  3. Kiểm tra dữ liệu đầu vào: kiểu dữ liệu, giá trị không âm, khối lượng tối thiểu (1 lô = 100 CP).
  4. Kiểm tra giá đặt lệnh nằm trong biên độ [giá sàn, giá trần] (gọi Market Data Service).
  5. Kiểm tra số dư ví khả dụng có đủ không (Giá × Khối lượng).
  6. Đóng băng (freeze) số tiền tương ứng trong ví user để tránh chi tiêu kép.
  7. Tạo bản ghi Order trong DB: order_id (UUID), user_id, ticker, type = BUY, price, quantity, status = PENDING, created_at.
  8. Publish event "ORDER_CREATED" lên Message Queue để Matching Engine xử lý bất đồng bộ.
  9. Trả về HTTP 201: order_id, status = PENDING, message = "Lệnh đặt thành công, đang chờ khớp".

Implementation Note

Source hiện tại (PortfolioController + PortfolioService) trả về một Transaction với HTTP 200; chưa có status PENDING, freeze ví, kiểm tra biên độ giá, hay event ORDER_CREATED. Các bước 3–8 của SRS là unspecified trong source.
