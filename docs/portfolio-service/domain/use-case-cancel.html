Use Case

UC-CANCEL — Cancel Order

UC-CANCEL · Updated 2026-06-02 · Source root services/portfolio-service

Investor cancels an outstanding PENDING order and reclaims frozen assets (documented requirement).

UC-CANCELuse-casepartial

Key Facts (SRS)

ActorUser (Nhà đầu tư), Manager, Admin (qua API Gateway)
GoalHủy một lệnh Mua/Bán đang ở trạng thái chờ khớp (PENDING)
TriggerUser nhấp "Hủy" cạnh lệnh đang ở trạng thái PENDING trên màn hình Sổ lệnh
Preconditions
  • User đã đăng nhập (JWT hợp lệ, chưa hết hạn).
  • Order Service hoạt động bình thường (health check 200 OK).
  • Message Queue (RabbitMQ) đang kết nối và sẵn sàng nhận message.
  • Database Orders đang kết nối và ghi được.
  • Lệnh cần hủy tồn tại và thuộc về user đang thao tác.
Postconditions (success)
  • Bản ghi lệnh cập nhật status = CANCELLED, cancelled_at = now().
  • Toàn bộ tài sản đóng băng (tiền hoặc cổ phiếu) được giải phóng về số dư khả dụng.
  • Event "ORDER_CANCEL_REQUESTED" đã được gửi đến Matching Engine qua Message Queue.
  • User nhận HTTP 200 với status = CANCELLED.
Postconditions (failure)
  • Không có bản ghi nào bị thay đổi trong database.
  • Không thay đổi số dư ví hoặc danh mục cổ phiếu của user.
  • User nhận HTTP 4xx/5xx kèm thông báo lỗi tương ứng.
Business FlowBFLOW-ORDER-EXECUTION
ExceptionsSRS Order Exception Flows (E1–E10)
TraceabilityTRACE-PORTFOLIO

Main Scenario (SRS)

  1. User nhấp "Hủy" cạnh lệnh đang ở trạng thái PENDING trên màn hình Sổ lệnh.
  2. Hệ thống kiểm tra JWT Token, xác thực và trích xuất userId.
  3. Truy vấn DB: tìm lệnh theo order_id, kiểm tra lệnh có thuộc về user này không.
  4. Kiểm tra trạng thái lệnh. Chỉ cho phép hủy lệnh có status = PENDING.
  5. Gửi yêu cầu hủy đến Matching Engine qua Message Queue (event "ORDER_CANCEL_REQUESTED").
  6. Cập nhật trạng thái lệnh trong DB: status = CANCELLED, cancelled_at = now().
  7. Giải đóng băng (unfreeze) số tiền/cổ phiếu đã bị khóa, trả về ví khả dụng cho user.
  8. Trả về HTTP 200: status = CANCELLED, message = "Hủy lệnh thành công".

Implementation Note

Source hiện tại không có endpoint hủy lệnh, không có vòng đời order (status/cancelled_at) và không có Matching Engine. Toàn bộ use case này là unspecified trong source và được giữ lại từ SRS.
