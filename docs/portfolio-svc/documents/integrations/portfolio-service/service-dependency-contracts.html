Minimum Market-Service Price Event Shape

Integration is fully event-driven via RabbitMQ. portfolio-service listens to price update messages from market-service to dynamically update its local price cache used for order validation and matching.

Exchange: market.exchange
Routing Key: price.updated

Payload:
{
  "symbol": "FPT",
  "price": 100.00,
  "close": 100.00,
  "latestPrice": 100.00,
  "timestamp": "2026-06-05T07:00:00Z"
}

Acceptance Checklist For End-To-End Completion

  • BUY rejects orders outside the market daily band and outside trading hours.
  • SELL rejects orders outside the market daily band and outside trading hours.
  • Write APIs derive user identity from gateway/auth, not directly from untrusted client body/query fields.
  • A filled order appends exactly one or more immutable transaction records and updates order status to FILLED.
  • A cancel request handles the race where an order is already filled before cancellation is processed.
  • Portfolio-updated events are versioned and consumed by ai-service without breaking payload compatibility.
  • PnL uses latest market prices or an explicitly documented fallback when prices are unavailable.
