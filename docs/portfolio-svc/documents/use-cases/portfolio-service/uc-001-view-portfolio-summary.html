UC-001: View Portfolio Summary

Service: portfolio-service Status: Partial

Use Case Specification (SRS)

AttributeDetail
NameView Portfolio Summary
Primary ActorUser (investor). Also visible to Manager and Admin.
Pre-conditionsThe caller is authenticated and a valid user id (UUID) is available (BR-001).
Main Flow 1. Caller sends GET /portfolio?userId={uuid}.
2. System loads the user's portfolio (cash and holdings).
3. System loads the holding list and transaction history.
4. System computes market value, total value, and unrealized PnL.
5. System returns the portfolio summary (API-001); total PnL can also be read via API-004.
Post-conditionsRead-only: the request returns portfolio data and changes no state.
Linked FRsFR-PORTFOLIO-01, FR-PORTFOLIO-02, FR-PORTFOLIO-08, FR-PORTFOLIO-09
Linked APIsAPI-001, API-004
