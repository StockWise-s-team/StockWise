UC-003: Place Sell Order

Service: portfolio-service Status: Implemented

Use Case Specification (SRS)

AttributeDetail
NamePlace Sell Order (Đặt lệnh bán)
Primary ActorUser (investor). Also visible to Manager and Admin.
TriggerUser enters order details (symbol, price, quantity) with side SELL and submits.
Pre-conditions1. Valid JWT.
2. portfolio-service healthy.
3. Broker connected.
4. Store writable.
5. Symbol within a trading session.
Main Flow 1. User submits symbol, type=SELL, price, quantity.
2. System validates JWT and extracts user id.
3. System validates input format and values.
4. System checks the price is within the valid band via Market Data (BR-100).
5. System checks available held quantity, not locked by another sell (BR-102).
6. System locks the corresponding shares to prevent double-selling.
7. System persists the order/transaction with type=SELL.
8. System publishes an order/transaction event (MSG-002).
9. System returns the order result.
Post-conditionsOn success: a SELL transaction row exists, shares are locked/reduced, and an event is published. On failure: nothing changes.
Linked FRsFR-PORTFOLIO-04, FR-PORTFOLIO-05, FR-PORTFOLIO-06, FR-PORTFOLIO-10
Linked APIsAPI-002
