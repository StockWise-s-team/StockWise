UC-002: Place Buy Order

Service: portfolio-service Status: Partial

Use Case Specification (SRS)

AttributeDetail
NamePlace Buy Order (Tiếp nhận lệnh mua)
Primary ActorUser (investor). Also visible to Manager and Admin.
TriggerUser enters order details (symbol, price, quantity) with side BUY and submits on the trading sandbox screen.
Pre-conditions1. User is authenticated with a valid JWT.
2. portfolio-service is healthy.
3. Message broker is connected.
4. The orders/transactions store is writable.
5. The symbol is within a trading session.
Main Flow 1. User submits symbol, type=BUY, price, quantity.
2. System validates the JWT and extracts the user id.
3. System validates input: types, non-negative values, minimum lot (BR-103).
4. System checks the price is within the daily band via Market Data (BR-100).
5. System checks available cash ≥ price × quantity (BR-101).
6. System freezes the corresponding cash amount.
7. System persists the order/transaction.
8. System publishes an order/transaction event (MSG-002).
9. System returns the order result.
Post-conditionsOn success: a transaction row exists, cash is reduced/frozen, an event is published, and the caller receives the order result. On failure: no row, no event, no balance change.
Linked FRsFR-PORTFOLIO-03, FR-PORTFOLIO-05, FR-PORTFOLIO-06, FR-PORTFOLIO-10
Linked APIsAPI-002
