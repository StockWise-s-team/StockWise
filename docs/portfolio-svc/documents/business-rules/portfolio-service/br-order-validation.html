BR: Portfolio Business Rules

Service: portfolio-service Source: StockWise SRS

Business rules the portfolio & paper-trading module must enforce, each with a simple implementation status.

IDRuleStatusApplies
BR-001 A userId supplied to a portfolio endpoint must be a valid UUID. Implemented API-001, API-002, API-004
BR-002 An order request carries userId, symbol, type, quantity, and an optional price. Implemented API-002, UC-002, UC-003
BR-009 Cash balance and holding quantity must never be negative (FR-PORTFOLIO-06). Partial ENTITY-001, ENTITY-002
BR-100 An order price must be within the daily band [floor, ceiling] obtained from Market Data. Not implemented UC-002, UC-003 (E1)
BR-101 A BUY order is rejected when available cash < price × quantity. Not implemented FR-PORTFOLIO-05 (E2)
BR-102 A SELL order is rejected when available (unlocked) holding quantity < order quantity. Not implemented FR-PORTFOLIO-05 (E3)
BR-103 Order input must be valid: correct types, non-negative values, minimum lot size. Not implemented NFR-USA-03
BR-104 Orders may only be placed during trading hours. Not implemented UC-002 (E10)
BR-105 Only a PENDING order owned by the caller may be cancelled; cancellation releases frozen assets. Not implemented UC-004 (E4, E5)

Data Constraints

  • Portfolio cash and holding quantity are non-negative (BR-009).
  • A holding is unique per (portfolio, symbol).
  • A transaction type is one of BUY / SELL, and quantity is greater than zero.
  • Transactions are append-only (FR-PORTFOLIO-07).

Status Legend

  • Implemented Enforced. Partial Partially enforced. Not implemented Required by SRS, not yet enforced.
