FR: Portfolio & Paper Trading

Service: portfolio-service Module: Portfolio & Paper Trading Source: StockWise SRS (báo cáo giữa kỳ, Nhóm 21)

Functional requirements for the Portfolio & Paper Trading module as specified in the StockWise software requirements specification. Each requirement carries a simple implementation status.

Status Legend

  • Implemented The behavior is available.
  • Partial The behavior is only partially available.
  • Not implemented Required by the SRS but not yet available.
IDSpecified RequirementStatus
FR-PORTFOLIO-01 Each user SHALL have a portfolio initialised with a virtual cash balance. Implemented
FR-PORTFOLIO-02 The system SHALL let a user view portfolio information: cash, holdings, market value, total value, and unrealized PnL. Implemented
FR-PORTFOLIO-03 The system SHALL let a user place a BUY order for a symbol with a quantity and price. Implemented
FR-PORTFOLIO-04 The system SHALL let a user place a SELL order for a held symbol. Implemented
FR-PORTFOLIO-05 The system SHALL reject an order when the user has insufficient cash (BUY) or insufficient holdings (SELL). Implemented
FR-PORTFOLIO-06 The system SHALL NOT allow negative cash balance or negative holding quantity. Implemented
FR-PORTFOLIO-07 The system SHALL store transactions as append-only (no update, no delete). Implemented
FR-PORTFOLIO-08 The system SHALL let a user view transaction history. Implemented
FR-PORTFOLIO-09 The system SHALL compute realized and unrealized PnL for the user. Implemented
FR-PORTFOLIO-10 After each transaction the system SHALL publish an event so the AI advisor has fresh portfolio context. Implemented
FR-PORTFOLIO-11 On receiving a price-update event the system SHALL recompute portfolio valuation / PnL. Implemented
