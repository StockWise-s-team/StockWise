Glossary: Portfolio & Paper Trading

Service: portfolio-service Source: StockWise SRS

Terms used across the portfolio-service documentation, drawn from the StockWise SRS glossary and scoped to portfolio and paper-trading concepts.

TermDefinition
Paper TradingSimulated trading using virtual cash, carrying no real financial risk.
PnLProfit and Loss. Realized PnL is the result of closed positions; unrealized PnL is the mark-to-market gain/loss on open holdings.
PortfolioA user's paper-trading account: a virtual cash balance plus the set of holdings owned.
HoldingThe quantity of a symbol a user currently owns, with an average purchase price.
CashThe virtual cash balance available in a portfolio (default 100,000,000).
SymbolA stock ticker code, e.g. FPT, VNM, HPG. Normalised to uppercase system-wide.
TransactionAn executed BUY or SELL record. Stored append-only (never updated or deleted).
OrderA user request to BUY, SELL, or CANCEL. An order has a lifecycle (e.g. PENDING, FILLED, CANCELLED) and, once executed, produces a transaction record.
JWTJSON Web Token. Stateless authentication mechanism verified by the API gateway.
RabbitMQMessage broker used for asynchronous communication between services (e.g. price.updated events).
OHLCOpen, High, Low, Close — price data points used by the market module; relevant to portfolio valuation inputs.
