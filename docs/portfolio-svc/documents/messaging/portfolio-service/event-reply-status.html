MSG-002: Portfolio Transaction Event

RabbitMQ Status: Not implemented

An outbound event published after each transaction so downstream consumers (e.g. ai-service) have fresh portfolio context, per FR-PORTFOLIO-10.

Specified Outbound Event

TriggerA transaction (BUY/SELL) is successfully recorded.
PurposeNotify downstream consumers that the user's portfolio changed.
Ordering rulePublish only after the database write commits (NFR-REL-02).
Payload compatibilityDo not change an existing payload without notifying consumers (NFR-SCA-02).

Related Events

EventPurpose
Portfolio-updated event (FR-PORTFOLIO-10)Inform ai-service of a portfolio change after a transaction.
Order lifecycle events (ORDER_CREATED / ORDER_CANCEL_REQUESTED)Drive matching and cancellation; depend on the order entity (ENTITY-003).
