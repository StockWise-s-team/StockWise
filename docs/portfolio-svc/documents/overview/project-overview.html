Project Overview: portfolio-service

Service: portfolio-service Module: Portfolio & Paper Trading Source: StockWise SRS (báo cáo giữa kỳ, Nhóm 21)

System Context

StockWise is a Vietnamese-market stock analysis and paper-trading platform built as microservices. It comprises five functional modules — Auth/User/Gateway, AI Advisor, Market Data & Chart, Portfolio & Paper Trading, and Data Pipeline — running on PostgreSQL, Qdrant, RabbitMQ, and a Next.js frontend, orchestrated by Docker Compose. This documentation set covers the portfolio-service, which implements the Portfolio & Paper Trading module.

Module Purpose

The portfolio-service lets each user manage a virtual-cash paper-trading account: view their portfolio (cash, holdings, transactions, PnL), place simulated BUY/SELL orders, cancel pending orders, and receive valuation updates as prices change.

Documentation Approach

This set documents the behavior specified in the StockWise SRS and tags each item with a simple implementation status: Implemented, Partial, or Not implemented.

Capability Status

  • Implemented View portfolio — GET /portfolio returns portfolio, holdings, and transactions.
  • Implemented Place order — POST /portfolio/order for BUY/SELL.
  • Implemented Read PnL — GET /portfolio/pnl.
  • Implemented Data model and runtime wiring for Portfolio, Holding, Transaction with PostgreSQL, RabbitMQ, and actuator.
  • Implemented Order validation, order cancellation, PnL calculation, and outbound transaction events.
  • Remaining Dependency Market price-band validation, matching/fill engine, and AI advisor event consumer.
